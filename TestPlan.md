<a name="introduction"/>

## Введение

Данный документ описывает план тестирования приложения "Project Management System". Документ предназначен для людей, выполняющих тестирование данного проекта. Цель тестирования - проверка соответствия реального поведения программы проекта и ее ожидаемого поведения, которое описано в требованиях.

<a name="items"/>

## Объект тестирования

В качестве объектов тестирования можно выделить следующие функциональные требования:

* Управление проектами (создание, редактирование, удаление, просмотр);
* Управление задачами (создание, редактирование, удаление, массовое создание);
* Навигация между проектами и задачами;
* Валидация данных форм;
* Работа с датами и сроками выполнения задач.

<a name="quality"/>

## Атрибуты качества

1. Функциональность:
    - функциональная полнота: приложение должно выполнять все заявленные функции;
    - функциональная корректность: приложение должно выполнять все заявленные функции корректно;
2. Удобство использования:
    - интуитивно понятный интерфейс на основе Material-UI с темной темой;
    - быстрый отклик на действия пользователя;
    - корректная валидация вводимых данных;
    - удобное управление датами через DateTimePicker.
3. Производительность:
    - время загрузки страниц менее 1 секунды;
    - время ответа API менее 300 мс.

<a name="risk"/>

## Риски

В данном случае к рискам можно отнести:
* Некорректная работа при отсутствии подключения к backend API;
* Проблемы с валидацией дат и сроков выполнения задач;
* Ошибки при массовом создании задач;
* Проблемы совместимости DateTimePicker с различными браузерами;
* Некорректное отображение темной темы в некоторых браузерах.

<a name="features"/>

## Аспекты тестирования

В ходе тестирования планируется проверить реализацию основных функций приложения. Аспекты, подвергаемые тестированию: 

### Управление проектами
* Создание нового проекта
* Редактирование существующего проекта  
* Удаление проекта
* Просмотр списка проектов
* Валидация обязательных полей

### Управление задачами
* Создание новой задачи в проекте
* Редактирование существующей задачи
* Удаление задачи
* Массовое создание задач
* Установка и изменение сроков выполнения
* Валидация данных задач

### Навигация и интеграция
* Переход от проекта к его задачам
* Связи между проектами и задачами
* Обработка ошибок API
* Responsive-дизайн на различных устройствах

### Создание проекта
Этот вариант использования необходимо протестировать на:
* успешное создание проекта с корректными данными;
* валидацию обязательных полей (название проекта);
* отображение сообщений об ошибках;
* корректное отображение формы в темной теме.

### Редактирование проекта
Этот вариант использования необходимо протестировать на:
* загрузку текущих данных проекта;
* успешное обновление данных;
* валидацию изменяемых данных;
* отмену редактирования без сохранения.

### Удаление проекта
Этот вариант использования необходимо протестировать на:
* подтверждение удаления через интерфейс;
* каскадное удаление связанных задач;
* обновление списка после удаления.

### Создание задачи
Этот вариант использования необходимо протестировать на:
* успешное создание задачи с корректными данными;
* валидацию обязательного поля названия задачи;
* работу DateTimePicker для установки сроков;
* обработку некорректных данных.

### Массовое создание задач
Этот вариант использования необходимо протестировать на:
* добавление нескольких задач одновременно;
* валидацию каждой задачи в массовой форме;
* удаление задач из массовой формы;
* обработку ошибок при массовом создании.

### Управление сроками задач
Этот вариант использования необходимо протестировать на:
* установку срока выполнения задачи;
* изменение срока выполнения;
* валидацию дат (нельзя установить прошедшую дату);
* корректное отображение дат в списке задач.

## Нефункциональные требования:
* Производительность: время загрузки страниц < 1 сек, время ответа API < 300 мс
* Совместимость: работа в Chrome, Firefox, Safari, Edge (последние 2 версии)
* Responsive-дизайн: корректное отображение на экранах 320px-4K
* Удобство использования: интуитивный интерфейс с темной темой

<a name="approach"/>

## Подходы к тестированию

При тестировании будет использован комбинированный подход:
* **Ручное тестирование** - для проверки UI/UX и основных пользовательских сценариев
* **Автоматизированное тестирование** - для регрессионного тестирования API endpoints
* **Тестирование валидации** - для проверки корректности работы форм
